INSTALLATION DE LA SUITE MEDINTUX par  Data Medical Design

Prérequis : L'installation de Medintux décrite ci-après suppose et nécessite:

  • Que vous soyez sous une distribution récente de Linux, Windows, Mac OSX 10.6 (minimum)
  • Que soit installé correctement MySQL 
    • sous Windows je vous conseille EasyPHP http://www.easyphp.org
    • sous Linux voir votre gestionnaire de paquets.
    • sous Mac mysql-5.5.10-osx10.6-x86.dmg
  • Que soit installé sous Linux Qt3 et Qt4 avec le support et drivers MySQL (voir votre gestionnaire de paquets) sous Mac et Windows les librairies sont déjà dans l'archive.

Pour une installation sous Windows.

les exécutables n'ont pas à être compilés, ils sont prêts a fonctionner : 

    ° Les noms utilisés, le sont à titre d'exemple.
    Dans cet exemple nous utilisons MedinTux-V2.14.7z comme nom, mais pour tout autre nom d'une autre version de MedinTux le fonctionnement est et sera similaire au nom près, qu'il vous faudra remplacer bien sûr dans l'exemple ci-dessous.

    ° Décompacter l'archive.
    Décompactez 'MedinTux-V2.14.7z' à l'endroit de votre choix, exemple 'C:\Program Files\' cela créera le répertoire : 'C:\Program Files\MedinTux-V2.14' contenant tout ce qu'il faut pour faire fonctionner MedinTux.

    ° Les fameuses DLL QT.
    Pour fonctionner correctement MedinTux se sert de superbes composants logiciels appelés Dll fournies par QT de Trolltech http://www.trolltech.com . Ces composants ou Dll de QT sont situées et fournies dans le répertoire 'MedinTux-V2.14/Qt' donc dans notre exemple 'C:\Program Files\MedinTux-V2.14\Qt'
    Windows a besoin de savoir où trouver ces Dll. Pour indiquer à Windows l'emplacement de ces Dll il nous faut renseigner avec l'emplacement des Dll de Qt, la variable d'environnement 'path' de Windows (un truc obscur que Windows consulte pour retrouver notamment les chemins des dll et programmes).

    ° Positionner la variable d'environnement  path  sur le répertoire créé par le décompactage.  
    Pour ce faire :


    En faisant sous XP: bouton droit sur poste de travail --> option menu "propriétés" --> Onglet "Avancé" -->
    bouton "Variables d'environnement" --> liste "Variables Système"  



    cliquez dans la liste la ligne commençant par : path
    bouton 'Modifier'  rajouter au début : 'C:\Program Files\MedinTux-V2.14\Qt;'   (sans les '') ou



    tout autre chemin d'installation de MedinTux si ce n'est pas celui-ci.  
    terminez en cliquant sur OK

    ° Exécuter le programme:   C:\Program Files\MedinTux-V2.14/bin/set_bases.exe
    se connecter à MySql
    et installer les bases de MedinTux


Pour une installation sous Linux.

    ° Les noms utilisés, le sont à titre d'exemple.
    Dans cet exemple nous utilisons MedinTux-V2.14.7z comme nom, mais pour tout autre nom d'une autre version de MedinTux le fonctionnement est, et sera similaire au nom près, qu'il vous faudra remplacer bien sûr dans l'exemple ci-dessous.

    ° Décompacter l'archive.
    Décompactez 'MedinTux-V2.14.7z' à l'endroit de votre choix, exemple '/home/moncompte/' cela crééra le répertoire : '/home/moncompte/MedinTux-V2.14' contenant tout ce qu'il faut pour faire fonctionner MedinTux.

    ° Exécuter set_bases
    Lancez le programme : /home/moncompte/MedinTux-V2.14/Programmes/set_bases/bin/set_bases si tout est ok et qu'il se lance, se connecter à MySql et installer les bases de MedinTux.
    sinon il vous faut vérifier si QT3 est bien installé, et peut être recompiler MedinTux.

    ° Recompiler MedinTux Placez vous dans le répertoire : /home/moncompte/MedinTux-V2.14/Programmes/
    A => ouvrir une console et exécuter : ./install.sh
    cela compilera et exécutera le programme d'installation set_bases dans lequel :
    B => Cliquez sur 'Compilation des exécutables' pour générer tous les
    programmes nécessaires.  
    C => se connecter à MySql et installer les bases de MedinTux

    ° Démarrer MedinTux
     cliquez sur le bouton: "Démarrer MedinTux" pour lancer l'exécutable principal.

Pour une installation sous Mac OSX 10.6.

    ° Les noms utilisés, le sont à titre d'exemple.
    Dans cet exemple nous utilisons MedinTux-V2.14.7z comme nom, mais pour tout autre nom d'une autre version de MedinTux le fonctionnement est, et sera similaire au nom près, qu'il vous faudra remplacer bien sûr dans l'exemple ci-dessous.

    ° Installer cette version (numéro de version minimum 5.5.10) de MySQL:

    ° Décompacter l'archive.
    Décompactez 'MedinTux-V2.14.7z' à l'endroit de votre choix, exemple '/Applications/' cela crééra le répertoire : '/Applications/MedinTux-V2.14' contenant tout ce qu'il faut pour faire fonctionner MedinTux sous Mac.

    ° Exécuter set_bases
    Lancez le programme : /Applications/MedinTux-V2.14/Programmes/set_bases/bin/set_bases.app si tout est ok et qu'il se lance, se connecter à MySql et installer les bases de MedinTux.

    ° Démarrer MedinTux
     Exécutez sur le programme : /Applications/MedinTux-V2.14/Programmes/Manager/bin/Manager.app pour lancer l'exécutable principal.

Configuration multipostes (CLIENT SERVEUR):

Lors d'une utilisation en multipostes, seul un poste: LE SERVEUR détient 
les bases de données et MySQL. Les autres postes CLIENTS se connectent 
à ce serveur via le réseau et pour eux l'installation de MySQL n'est pas nécessaire. 

PIEGES    Pensez à ouvrir les Firewalls  sur les adresses IP des serveurs et clients, port: 3306
Sous XP SP2 il est fermé ( pas sur le SP1).

Chaque CLIENT se connecte au serveur MySQL par:

° un nom de connexion : exemple 'poste_secretariat' 

° le nom du serveur auquel il doit se connecter: ce nom est 

    soit le nom de machine du serveur si il est défini: exemple : 'serveur_medintux'
    soit tout simplement l'adresse IP du serveur: exemple:  192.168.0.5


° un mot de passe de connexion

COTE SERVEUR:     
     
Pour chaque poste client devant se connecter au serveur, 
créer SUR LE SERVEUR un utilisateur MySQL avec les droits suivants 
d'écriture et lecture des bases MySql suivantes: 

    DrTuxTest        (lecture écriture)
    MedicaTuxTest    (lecture écriture)
    BaseGetTest      (lecture écriture)
    CIM10Test        (lecture )
    CCAMTest         (lecture )
    QLaboFTPTest     (lecture écriture)
    DatasempTest     (lecture écriture)   si vous avez installé le Vidal Datasemp


° définir aussi si besoin le mot de passe de connexion pour le client.
° L'adresse IP de cette machine cliente exemple:  192.168.0.3     
 
COTE CLIENT:

Méthode cliquodrome: Renseignez les champs:

    Nom de connexion:  (nom de votre machine) exemple: serveur_medintux
    Mot de passe:      (si vous en avez défini un pour cet utilisateur MySQL)
    Nom ou adresse IP de la machine serveur MySQL distante: (adresse IP du serveur MySQL) exemple: 192.168.0.5
    CLiquez sur "Appliquer"
